# MLO Installation

Below I'll show you how to properly pack MLOs. You'll be able to stream almost all MLOs into one single resource rather than streaming different ones.

{% hint style="success" %}
You can download a template by clicking [here.](https://cdn.discordapp.com/attachments/983974550289154098/1009128769694474302/assets_map.rar) \
\
They are empty folders with nothing inside, just named to facilitate this guide.&#x20;
{% endhint %}

{% hint style="danger" %}
You cannot pack MLOs that are encrypted, like Gabz map. You can tell if they're encrypted if the folder comes with a .fxap&#x20;
{% endhint %}

### Using the template above for as reference

{% hint style="success" %}
All of your stream files should go on the stream folder. Inside each of the folders, you'll place **ALL** of your stream files. \
\
You can also place .ymap extensions on these stream folders.
{% endhint %}

![Stream Files](<../../../.gitbook/assets/image (111).png>)

{% hint style="danger" %}
The MOST important rule about packing your MLOs is to make sure that your file name <mark style="color:red;">**\_manifest.ymf**</mark> is NEVER NAMED the same in all of your stream folder. \
\
I would recommend changing them to what the MLO is, for example, _\_manifestburgershot.ymf_, _\_manifestmrdp.ymf_, etc...
{% endhint %}
