<figure><img src="../../../../.gitbook/assets/image (271).png" alt=""><figcaption></figcaption></figure>

### Enable Notifications for Queue Events

Notify users via Discord Direct Message (DM) or in a Discord Channel in your server, when they join the queue or reach position #1

Enable the form of notification you would prefer (DM or channel message), or both, for either Queue event:

A) Queue Joined

### Application Notifications

{% stepper %}
{% step %}
### Application Submitted

Notify users about application submissions.&#x20;

This can be used to notify admins of application submissions to review by sending notifications to a submissions channel, or DMing a success notification to a player to inform them their submission was received.

<figure><img src="../../../../.gitbook/assets/image (274).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
### Status Updates

Notify users about application status updates, such as being accepted or rejected.

<figure><img src="../../../../.gitbook/assets/image (276).png" alt=""><figcaption></figcaption></figure>
{% endstep %}
{% endstepper %}

### Ticket Notifications

{% stepper %}
{% step %}
### Ticket Created

Notify users about ticket creation.

This can benefit admins by being notified when new tickets come in to address, and players to keep track of the status of their tickets.

<figure><img src="../../../../.gitbook/assets/image (277).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

Keep track of ticket updates by notifying users of ticket replies in real time.
