# Applications Builder

### Application System Overview

Create Application forms (like Job Postings on Linkedin,etc) for community members to fill in and submit applications for positions or roles such as, staff, EMS, Gang creation, Law Enforcement, Business creation, etc.

Application Submissions can be managed by staff to either be accepted or rejected.

Application Builder Features:

<details>

<summary>Auto-Assign Discord Roles on Approval</summary>

Grant Discord roles when an application is approved.

Select one or multiple roles from your Discord server.

Roles are assigned instantly upon approval.

</details>

<details>

<summary>Application Questions</summary>

Question Types Supported:

* Short Text
* Long Text
* Single Choice
* Multiple Choice

Answer Limits (Text Questions)

* Set minimum and maximum character length for text-based answers.

Choice Options (Choice Questions)

* Add/remove options for Single Choice and Multiple Choice questions.

</details>

<details>

<summary>Required Questions</summary>

Mark any question as required to submit the application.

</details>

### Create Application Forms

To create an Application, click "**Create Application**"

<figure><img src="../../../../.gitbook/assets/image (113).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
### Craft Application Specifications

Fill in the Application details:

* **Title** \[Title of Job Posting]
* **Description** \[Job Description]
* Allow multiple submissions from the same user (enable/disable)
* **Discord Roles to Assign on Approval (example: EMS, Police, etc)**
* **Questions**

**Example for application to gain the Whitelist Role:**

<figure><img src="../../../../.gitbook/assets/image (114).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
### Add Questions!

Click "+Add Question", there is no limit on questions!

<figure><img src="../../../../.gitbook/assets/image (115).png" alt=""><figcaption></figcaption></figure>

Configure the questions:\
\- Type (long/short answer, single/multiple choice)\
\- Label (the question itself)\
\- Required (check if you require this question answer, leave unchecked if optional)

Once you are satisfied with your Application form, click "**Create Application**"

<figure><img src="../../../../.gitbook/assets/image (116).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
### Editing or Deleting an Application

If things have changed, or you made a mistake on an application and need to go back and fix it, instead of creating another simply click "**Edit**" (pencil icon).

<figure><img src="../../../../.gitbook/assets/image (118).png" alt=""><figcaption></figcaption></figure>
{% endstep %}

{% step %}
### Share Application

A) Copy the Applications URL and share it with your community for them to submit their applications (example, hyperlinking the application in an announcement on Discord).

**OR**

B) They can also discover open Applications organically by going to the Applications section in the Landing Page.

Once they submit their application it will populate in their Applications section.

* Applications can be filtered by their Status (Pending, Accepted, Rejected, All) or by their ID.

<figure><img src="../../../../.gitbook/assets/image (120).png" alt=""><figcaption></figcaption></figure>
{% endstep %}
