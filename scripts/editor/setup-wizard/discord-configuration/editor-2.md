# Moderators

{% stepper %}
{% step %}
#### Adding new Moderators

In the Dashboard, enter the Moderators menu.

<figure><img src="../../../../.gitbook/assets/image (151).png" alt=""><figcaption></figcaption></figure>

Add a new moderator to help manage your server. They **must** already be a member of your Discord server.

Moderators are added using their **Discord User ID or Role.**

* Creating a Mod using the User ID option allows you to Mod specific members of your staff/Discord.
* Creating Moderators using the Role option allows you to apply Mod status to all members who are assigned a specific role in your Discord (like a Staff role, Admin/Mod role, etc)

{% step %}
#### Permissions

Ticket and application permissions are managed here. You can select one of the two or both, depending on what you want that user (mod created via UserID) or role (mods created via Role) to be able to manage.

<figure><img src="../../../../.gitbook/assets/image (222).png" alt=""><figcaption></figcaption></figure>

For example: Staff members could support higher-up Moderators by handling tickets, replying to users, and similar tasks. Moderators could manage tickets in addition to applications including approving, denying, and deleting them.

If you forgot what the user can manage, there are tags in their profile to indicate their permissions. You can click "Edit" to change these permissions, just remember to save\![](http://127.0.0.1:30120/1of1-webqueue/api/health)

{% hint style="danger" %}
If you want to be a Moderator, you’ll need to assign yourself the role. You don’t automatically get moderator permissions just because you own the web queue, you must explicitly grant yourself access either by assigning the moderator role or adding your user ID.
{% endhint %}
